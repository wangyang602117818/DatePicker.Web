Init
In the above samples，datepicker.min.js init Input-Tags always happens after the page is loaded, but when you modified pages manually, you need re-init datepicker.
1) Example One: if changed date-val value through javascript and you need show the value to front pages，you need call datepicker() method.
 Sample:
<script>
  $(function () {
   $("#txt1").attr("date-val", "2017-03-16 13:05:15").datepicker();
  });
</script>
<body>
  <input type="text" name="txt" id="txt1" class="datepicker" />
</body>
2) Example Two: the Input-Tag was appended to the page entirely through javascript, then call datepicker() method init it.
 Sample:
<script>
  $(function () {
   document.body.innerHTML = "<input type=\"text\" class=\"datepicker\" />";
   datepicker();
  });
</script>
<body>
</body>
